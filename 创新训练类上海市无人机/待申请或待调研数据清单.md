# 待申请或待调研数据清单

以下数据通常无法通过公开网页直接完整下载，建议作为下一阶段重点对接内容。

## 1. 无人机配送现状数据

- 现有无人机配送站点、起降点、充/换电站坐标
- 服务范围
- 日均配送量
- 班次、时段、运营频次
- 典型航线

建议获取渠道：

- 京东物流、顺丰、丰翼、美团等企业访谈或数据申请
- 上海低空经济相关试点园区、区级政府公开材料
- 新闻报道与企业白皮书做补充核对

## 2. 无人机技术参数数据

- 不同机型的续航、载荷、巡航速度、抗风等级、起降条件、电池更换时间

建议获取渠道：

- 企业官网技术规格页
- 产品手册、白皮书
- 学术论文与行业报告交叉校验

## 3. 低空空域约束数据

- 禁飞区
- 限飞区
- 临时空域管制范围
- 低空航路或试点走廊

说明：

- 这类数据通常不公开提供完整 GIS 文件。
- 公开网页常见的是规则说明，而不是可直接建模的矢量边界。

建议获取渠道：

- 民航、空管、公安、应急等部门公开文件
- 无人机平台公开飞行规则页
- 与低空经济试点单位或主管部门联系

当前已补充的可直接使用代理层：

- `05_低空空域约束数据/上海_低空约束_机场与起降点.gpkg`
- `05_低空空域约束数据/上海_低空约束_军事用地.gpkg`
- `05_低空空域约束数据/上海_低空约束_保护地.gpkg`

这些数据适合前期建模约束，但不等同于官方完整禁飞区/限飞区矢量库。

## 4. 动态运行数据

- 实时交通拥堵
- 历史交通速度
- 实时天气 API 长期序列

说明：

- 天气公开 API 较容易获得。
- 实时交通通常需要高德、百度、腾讯、滴滴、TomTom 等平台的开发者 Key 或商业授权。

建议获取渠道：

- Open-Meteo、气象局公开接口
- 高德开放平台、百度地图开放平台

## 5. 灾害与应急专题数据

- 历史洪涝点位
- 影响范围
- 风险等级
- 正式应急避难场所清单
- 城市应急救援通道

说明：

- 医院、消防、社区中心等可先用 OSM 近似替代。
- 真正高质量的洪涝风险面与应急通道数据更适合从政府部门或专业平台申请。

建议获取渠道：

- 上海市应急管理、规划资源、水务、气象等部门公开资料
- 学术数据库、灾害案例资料、政府报告

当前已补充的可直接使用基础层：

- `07_灾害与应急数据/上海_应急避难点_OSM.gpkg`
- `07_灾害与应急数据/上海_警务点_OSM.gpkg`
- `07_灾害与应急数据/上海_应急直升机起降点_OSM.gpkg`
- `07_灾害与应急数据/上海_水体面_洪涝相关基础层.gpkg`
- `07_灾害与应急数据/上海_河道水系_洪涝相关基础层.gpkg`

这些文件适合作为应急设施和洪涝相关基础底图，但不等同于官方历史洪涝点位、风险等级面或正式应急通道数据库。

## 6. 政策与制度数据

- 上海低空经济政策
- 城市规划与国土空间规划
- 空域管理规则
- 应急管理与防汛预案

说明：

- 这类文件能通过官网逐份下载，但需要花时间筛选。
- 后续建议单独做一个“政策文件库”，按年份、发文机关、主题分类。

## 7. 调研数据

- 企业访谈纪要
- 街道/社区访谈纪要
- 居民问卷
- 实地核查记录
- 站点候选位置照片

说明：

- 这部分只能通过你们项目组自行调研形成。

## 8. 成本效益数据

- 站点建设成本
- 场地租赁成本
- 运维成本
- 电池与充换电成本
- 人工调度成本
- 应急场景社会效益估算

建议获取渠道：

- 企业访谈
- 行业报告
- 招投标公告
- 学术论文中的案例参数

## 当前可行的替代策略

1. 用 `shanghai.gpkg` 先搭底图和需求空间分布。
2. 用 OSM 中的医院、消防、社区中心、学校、避难相关点位做应急设施近似。
3. 用代表性无人机机型参数先构建原型模型。
4. 把“企业运营数据”和“空域数据”作为后续申请数据，先完成方法框架和可运行 demo。
